# Інструкція з розгортання Азбука-Помічник

Цей проект налаштований для запуску у Docker контейнері. Ви можете розгорнути його на своєму сервері Ubuntu за допомогою `docker-compose` або Portainer.

## Передумови
- Встановлений Docker та Docker Compose.
- (Опціонально) Portainer для зручного керування.
- API Key від Google Gemini.

## Варіант 1: Розгортання через термінал (docker-compose)

1.  **Клонуйте репозиторій на сервер:**
    ```bash
    git clone <URL_ВАШОГО_РЕПОЗИТОРІЮ> abc-helper
    cd abc-helper
    ```

2.  **Створіть файл `.env` з вашим ключем:**
    ```bash
    echo "API_KEY=ваш_ключ_тут" > .env
    ```

3.  **Запустіть контейнер:**
    ```bash
    docker-compose up -d --build
    ```

4.  **Відкрийте у браузері:**
    Перейдіть за адресою `http://IP_ВАШОГО_СЕРВЕРА:3030` (наприклад, `http://192.168.50.249:3030`).

## Варіант 2: Розгортання через Portainer (Stacks)

Якщо ви хочете, щоб Portainer сам зібрав образ із вашого Git-репозиторію:

1.  Зайдіть у Portainer -> **Stacks** -> **Add stack**.
2.  Виберіть **Repository**.
3.  Введіть назву (наприклад, `abc-helper`).
4.  Введіть URL вашого Git-репозиторію.
5.  У розділі **Environment variables** додайте змінну:
    -   Name: `API_KEY`
    -   Value: `ваш_ключ_тут`
6.  Натисніть **Deploy the stack**.

Portainer завантажить код, збілдить Docker-образ і запустить його на порту 3030.

## Варіант 3: Локальна збірка на сервері + Portainer

Якщо ви вже скнлонували код на сервер у папку `/home/nicks/abc-helper`:

1.  Зайдіть у папку проекту:
    ```bash
    cd /home/nicks/abc-helper
    ```
2.  Збілдить образ вручну (якщо хочете):
    ```bash
    docker build -t abc-helper:local .
    ```
3.  Або просто запустіть стек через Portainer, обравши метод **Web editor** і вставивши вміст `docker-compose.yml`. Але для цього краще використовувати **Варіант 1** або **Варіант 2**.

## Оновлення

Щоб оновити додаток після змін у коді:
- **Через термінал:** `git pull && docker-compose up -d --build`
- **Через Portainer:** Натисніть кнопку **Update the stack** (якщо використовуєте Git repository stack).
